# Newsroom Prioritizer

Сервис приоритизации инфоповодов для редакции: собирает новости из открытых лент,
убирает дубли, оценивает вероятность «залетит / не залетит» и показывает редактору
ленту по убыванию приоритета. Помогает при ограниченном времени отобрать инфоповоды,
которые наберут больше просмотров.

> Статус: в разработке. Сейчас поднимается инфраструктура (шаг 0 из плана сборки).

## Архитектура

| Сервис | Роль |
|---|---|
| `api` (FastAPI) | REST API: приём инфоповодов, лента, скоринг |
| `web-proxy` (nginx) | реверс-прокси, единая точка входа |
| `webview` (Streamlit) | витрина редактора |
| `rabbitmq` | очередь задач для рерайта (медленный LLM вне HTTP) |
| `database` (PostgreSQL) | хранилище инфоповодов и обратной связи |

## Быстрый старт

```bash
cp .env.example .env        # при необходимости поправить значения
docker compose up --build
```

После старта доступны:

- Витрина (Streamlit): http://localhost:8501
- REST API (Swagger): http://localhost/docs
- Healthcheck: http://localhost/health
- Панель RabbitMQ: http://localhost:15672

## Переменные окружения

Все настройки — в `.env` (шаблон с комментариями: `.env.example`).
