# Запуск и проверка

Сервис разворачивается одной командой в Docker. Возможны два режима:

- **Быстрый (самодостаточный)** — для ознакомления/проверки. Ничего, кроме Docker, не нужно:
  внешние сервисы (LLM, Text.ru, Метрика) при пустых ключах аккуратно пропускаются, а приоритизатор
  без обученной модели работает на прозрачной эвристике. Всё поднимается и демонстрирует полный поток.
- **Полный** — с обученной моделью (через DVC) и внешними ключами.

## Требования

- Docker и Docker Compose (Docker Desktop на Windows/Mac либо Docker Engine на Linux).
- Доступ в интернет (сервис собирает новости из открытых RSS-лент).

## Быстрый запуск (в т.ч. для проверки жюри)

```bash
git clone https://github.com/tari-lucy/newsroom-prioritizer.git
cd newsroom-prioritizer
cp .env.example .env          # значений по умолчанию достаточно
docker compose up --build
```

Первый запуск скачивает образы (PostgreSQL, RabbitMQ, nginx, Python) — пара минут. После старта:

| Что | Адрес |
|---|---|
| Витрина редактора | http://localhost:8501 |
| REST API (Swagger) | http://localhost/docs |
| Healthcheck | http://localhost/health |
| Панель очереди RabbitMQ | http://localhost:15672 |

> На удалённом сервере вместо `localhost` — IP сервера; порты **8501** и **80** должны быть открыты.

## Как проверить, что всё работает

1. Открыть витрину http://localhost:8501, войти: **логин `editor`, пароль `editor123`**.
2. Нажать **«🔄 Запустить сбор»** — сервис соберёт свежие инфоповоды из лент Севастополя/Крыма.
   В сводке видно: собрано / новых / дублей / вне региона.
3. Вкладка **«Лента»** — карточки инфоповодов, отсортированные по вероятности «залетит»,
   с классом приоритета (🔴/🟡/⚪) и гео-тегами. Проверяется гео-фильтр (чужие регионы отсеяны),
   дедуп и скоринг.
4. На карточке — **👍/👎** (обратная связь копится как обучающий сигнал) и **«✍️ Рерайт»**
   (готовит переписанный текст; без ключа LLM — черновик-заглушка).
5. Вкладка **«Источники»** — добавить/выключить ленту через интерфейс.

Проверка через API (Swagger `http://localhost/docs`): `POST /auth/login` (editor/editor123) →
кнопка Authorize → `POST /ingest` → `GET /feed`.

## Полный режим (обученная модель и внешние сервисы)

1. **Обученная модель** (иначе работает эвристика-заглушка). Модель версионируется через DVC:
   ```bash
   pip install "dvc[ssh]"
   dvc pull                 # подтянет config/virality_logreg.joblib из хранилища
   ```
2. **Ключи внешних сервисов** — вписать в `.env` (пустые = функция пропускается):
   - `LLM_API_KEY`, `LLM_MODEL` — рерайт через vsellm;
   - `TEXTRU_API_KEY` — проверка уникальности рерайта;
   - `METRIKA_TOKEN`, `METRIKA_COUNTER` — просмотры как метки для переобучения.
3. Перезапустить: `docker compose up --build`.

## Остановка

```bash
docker compose down          # остановить (данные в БД сохранятся в volume)
docker compose down -v       # остановить и удалить данные
```

## Если что-то не поднялось

- Посмотреть логи сервиса: `docker compose logs api` (или `webview`, `worker`, `scheduler`, `trainer`).
- Проверить, что порты 80/8501 не заняты другими программами.
- Лента пуста — нажать «Запустить сбор» ещё раз (лентам нужен доступ в интернет).
